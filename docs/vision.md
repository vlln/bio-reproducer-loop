## 一、业务目标

解决 bio-reproducer 当前缺乏系统化测评能力的核心问题：无法量化系统性能、无法追踪改动效果、无法与同类系统比较。

构建五层测试与基准体系（L1-L5），从单 Agent 可靠性到真实环境端到端复现，覆盖开发迭代和公开测评两个场景。

L1-L2 为内部测试（耦合于 bio-reproducer 实现），L3-L5 为黑盒基准（引擎无关，可成为公开标准）。

---

## 二、用户范围

- bio-reproducer 开发者：在开发迭代中快速验证改动效果
- 生物信息学研究者：评测复现工具链的表现
- Agent 系统开发者：评测 Agent 在科学复现任务上的能力
- 基准社区：使用公开基准对比不同复现系统

---

## 三、长期理想形态

1. 开发者修改 prompt 后，L1/L2 测试在 CI 中自动运行，分钟级反馈
2. L3 构造论文基准可频繁运行，追踪系统能力变化
3. L4 工程基准（冻结环境的真实论文）定期运行，产出可比较的复现率报告
4. L5 生产基准持续采样，监控真实世界复现率
5. L3-L5 基准可独立发布，被其他复现系统采用
