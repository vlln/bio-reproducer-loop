## 一、项目组成

bio-reproducer 包含三个部分：

1. **论文复现系统** — AI Agent 驱动的 7 阶段工作流，输入论文 PDF/DOI，自动完成信息提取、环境搭建、数据获取、分析运行、结果验证
2. **内部测试（L1-L2）** — 耦合于复现系统的实现，验证单 Agent 业务逻辑和跨 Phase 信息流
3. **公开 benchmark（L3-L5）** — 引擎无关的论文复现基准，可独立发布，被任何复现系统采用

---

## 二、业务目标

### 复现系统

让研究者输入一篇论文，系统自动完成复现全流程，产出可复现的复现包和结构化验证报告。降低生物信息学论文复现的门槛和成本。

### 内部测试

为复现系统的开发迭代提供快速反馈——修改 prompt 或 workflow 后，能立刻知道 Agent 的行为是否退化。

### 公开 benchmark

建立可量化、可比较、可独立发布的论文复现测评标准，使不同复现系统可以在同一基准上对比。

---

## 三、用户范围

- 生物信息学研究者：复现论文、评测复现工具链
- bio-reproducer 开发者：在开发迭代中验证改动效果
- Agent 系统开发者：评测 Agent 在科学复现任务上的能力
- 基准社区：使用公开基准对比不同复现系统

---

## 四、长期理想形态

### 复现系统

用户输入论文 DOI，系统自动完成全流程复现，产出可独立运行的复现包（run.sh + README + 验证报告）。

### 内部测试

开发者修改 prompt 后，L1/L2 在 CI 中自动运行，分钟内反馈 Agent 行为变化。

### 公开 benchmark

1. L3 构造论文基准频繁运行，追踪系统能力变化
2. L4 工程基准（冻结环境的真实论文）定期运行，产出可比较的复现率报告
3. L5 生产基准持续采样，监控真实世界复现率
4. L3-L5 基准可独立发布，有自己的版本号，被其他复现系统采用
